# Code Review Checklist

在进行完整代码评审时，使用这份清单确保关键维度都覆盖到。

---

## 1. Security

- [ ] 是否存在认证或授权绕过风险？
- [ ] 是否有敏感数据暴露、日志泄露或调试信息外泄？
- [ ] 是否存在注入风险（SQL、命令、模板、XSS 等）？
- [ ] 输入是否经过校验和清洗？
- [ ] 密钥、token、凭据是否被硬编码？

## 2. Performance

- [ ] 是否存在明显的低效算法或不必要的嵌套循环？
- [ ] 是否可能出现 N+1 查询？
- [ ] 是否有重复计算可以缓存？
- [ ] 内存使用是否合理？
- [ ] 是否存在竞态条件、锁竞争或并发风险？

## 3. Code Quality

- [ ] 命名是否清晰、贴合领域语义？
- [ ] 是否遵循单一职责原则？
- [ ] 是否存在重复代码可以抽取？
- [ ] 错误处理是否清晰且一致？
- [ ] 文档、注释和类型信息是否足够？

## 4. Maintainability

- [ ] 函数是否过长或职责混杂？
- [ ] 类/模块是否过大？
- [ ] 依赖关系是否过于耦合？
- [ ] 代码是否容易测试和扩展？
- [ ] 是否有明显的技术债标记（TODO、FIXME、临时绕过）？

## 5. Testing

- [ ] 新逻辑是否有测试覆盖？
- [ ] 是否覆盖了关键分支和错误路径？
- [ ] 测试是否验证行为而不是实现细节？
- [ ] 是否缺少回归测试？
- [ ] 现有测试是否仍然可信、稳定？

## 6. Review Output

- [ ] 先给出最严重的问题
- [ ] 每个问题都包含位置、影响和修复建议
- [ ] 对高风险项明确说明严重级别
- [ ] 如果没有发现问题，明确说出“未发现问题”
- [ ] 标出剩余风险或测试缺口
